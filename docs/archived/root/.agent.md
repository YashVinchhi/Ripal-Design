# ProdCleaner Agent

name: ProdCleaner
version: 1.0
maintainer: GitHub Copilot (agent)

summary: |
  Agent to prepare this PHP webapp repository for production by:
  - creating a `Prod` branch from `ripal-design`
  - moving secrets to environment variables
  - archiving or removing non-production docs and "slope"/experimental code
  - running basic sanity checks and producing a clean, deployable branch

when_to_use: |
  Invoke this agent when you want a focused cleanup pass to produce a production-ready branch. Use it instead of the default agent for repository housekeeping and secrets hardening.

scope:
  - language: PHP
  - activities: repo-cleanup, secrets-hardening, archive-docs, branch-management

tools_allowed:
  - file-editing
  - git (create branch, commit)
  - run terminal commands
  - search (grep)

tools_restricted:
  - external network fetches
  - making destructive deletions without confirmation (agent will ask for confirmation on major deletes)

preferences:
  - Keep `README.md` and essential deploy docs.
  - Move other long-form `.md` files to `docs/archived/` by default (can delete on user approval).
  - Replace hardcoded credentials with `getenv(...)` calls and provide a `.env.example`.

example_prompts:
  - "Prepare a production-ready branch: create `Prod` from `ripal-design`, move secrets to env, archive docs, remove slope code."
  - "Run the ProdCleaner: audit for hardcoded DB and SMTP credentials and apply fixes."

notes: |
  - The agent will ask clarifying questions if it finds ambiguous items like what counts as "slope" code or which docs to keep.
  - Sensitive values will be moved to environment variables and a `.env.example` will be committed; `.env` is added to `.gitignore`.
